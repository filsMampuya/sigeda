# Suivi des iterations SIGEDA

Ce dossier contient les resumes de travail produits a chaque iteration significative du projet.

Objectifs :

- aider l'utilisateur humain a comprendre l'evolution du projet;
- garder une trace claire des decisions techniques et metier;
- faciliter la reprise du contexte par un assistant ou un developpeur;
- distinguer ce qui est fait, ce qui reste a valider et ce qui peut venir ensuite.

Avant chaque nouvelle phase de travail, lire :

1. `PROJECT_CONTEXT.md`
2. `CURRENT_STATE.md`
3. le dernier fichier dans `docs/iterations`

Apres chaque iteration significative, creer un nouveau fichier date dans ce dossier.
